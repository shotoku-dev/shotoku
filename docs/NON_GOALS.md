# Non Goals

Shotoku is not:

- A wallet
- A payment processor
- A payment network
- A custody provider
- A banking product
- A procurement platform
- An ERP
- A workflow engine
- A compliance suite (Shotoku may help teams produce audit trails, but it does not claim regulatory compliance by itself)
- An accounting platform

## Focus

Shotoku focuses on one question:

> Should this agent be allowed to perform this action?

Everything else is secondary.

## Design principle

If a feature does not improve:

- Authorization
- Approvals
- Policy enforcement
- Auditability

it should probably live outside Shotoku.
